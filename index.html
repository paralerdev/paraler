Your daily dev servers. All of them.

All your projects. All your services. One TUI. Start working — everything's up. Close paraler — everything stops. No more "which tab was that backend?"

Features

Start/stop everything One key to start, one to stop. S/X for all at once.
All logs, one place stdout and stderr from every service. Filter and export.
Health checks HTTP endpoints and port monitoring.
Service dependencies Start in order. Backend before frontend.
Auto-restart on crash Service died? Comes back automatically.
Hot config reload Edit yaml, Ctrl+R. No restart needed.

Right tool for the job

● Use paraler for

  • Dev servers — npm run dev, go run, cargo watch
  • Multiple projects — switch context without closing tabs
  • Local development — your machine, your workflow
  • Services you restart often — frontend, backend, workers
  • Watching logs — all output in one place

○ Not the right fit for

  • Databases, Redis — Docker does this better
  • Production servers — use systemd, k8s, pm2
  • Long-running infra — stuff that runs 24/7
  • CI/CD pipelines — not for automation
  • Single-service projects — just use your terminal

Configuration

One YAML file. Put it in your project root or ~/.config/paraler/. Auto-detects NestJS, React, Vue, Go, and more.

paraler.yaml
projects:
  client-portal:
    path: ~/work/client-portal
    services:
      api:
        cmd: npm run start:dev
        port: 3010
      web:
        cmd: npm run dev
        depends_on: [api]

  admin-dashboard:
    path: ~/work/admin
    services:
      backend:
        cmd: go run .
        port: 3020
      frontend:
        cmd: pnpm dev

Keybindings

Navigation: ↑/k up │ ↓/j down │ Tab switch panel │ pgup/pgdn scroll
Services: s start │ x stop │ r restart
Bulk: S start all │ X stop all │ v select │ V clear selection
Logs: / filter │ c clear │ e export │ g top │ G bottom
Projects: a add │ d delete service │ D delete project
Other: ctrl+r reload config │ ? help │ q quit
